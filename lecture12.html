Lecture 12: Intro to Pandas

Pandas is NumPy with labels: real tables with named columns and an index. This lesson is ordered by your homework, so every tool a question needs gets taught first, with examples you can edit and rerun, then that question is the practice.

>>> import pandas as pd

Why this exists

From NumPy to Pandas

NumPy gave you one block of numbers, all the same type. Real data is messier: named columns, mixed types, missing values, text mixed with numbers. Pandas is built on top of NumPy to handle exactly that.

  • Series, a single labeled column.
  • DataFrame, a whole table, which is really a bunch of Series sharing one index.
  • Everything has an index (the row labels) and column names, so you select by name, not just position.

.loc vs .iloc
.loc uses the label (what you see in the index), .iloc uses the integer position. When the index is just 0, 1, 2 they look the same, but they are not once the labels change.

Picking rows and columns

      
Selecting and masking

      
One column name in square brackets gives a Series. A list of names gives a smaller DataFrame. And a condition inside the brackets keeps only the rows where it is True, exactly like NumPy masking but on a whole table.

Splitting text into columns

      
.str.split

      
The .str accessor runs string methods down a whole column. .str.split("|", expand=True) splits each value on the bar and, with expand=True, spreads the pieces into separate columns. Rows with only one piece get NaN in the second column, which you can fill.

Handling blanks

      
fillna and dropna

      
fillna replaces missing values. The trick you need: you can fill from another column by passing that column, and Pandas lines them up by index. dropna(subset=[...]) drops the rows that are still missing in a given column.

Math on columns

      
Column arithmetic, sum, and round

      
A whole column has .sum(), .mean(), and so on, just like NumPy. Arithmetic between columns happens row by row. .round(0) rounds but keeps the values as floats.

Tools for Q3

      
groupby with an aggregate

      
groupby splits rows into groups by a column's value, then you apply an aggregate to each group. The bracket style decides the shape you get back, and Q3 needs a DataFrame.

      
single vs double brackets after groupby
groupby("x")["y"].mean() gives a Series. groupby("x")[["y"]].mean() gives a DataFrame. The double brackets keep it a table.

Cleaning tools

        
df["c"].str.split("|", expand=True)   split text into columns
s.fillna("None")                      replace blanks with a value
df["co2"].fillna(df["avg"])           fill blanks FROM another column
df.dropna(subset=["co2"])             drop rows blank in that column
df["c1"].str.contains("Boeing 737")   True/False for holding a phrase
df[~mask]                             keep rows where the mask is False
df["c"].sum()                         whole-column total
(rate * df["dur"]).round(0)           column math, rounded, stays float
df.to_csv("out.csv")                  write the table (index included)

        
Summaries and groups

some_series.mean()                    average a Series
round(value, 3)                       trim to 3 decimals
df.groupby("x")["y"].mean()           per group -> a Series
df.groupby("x")[["y"]].mean()         per group -> a DataFrame
df["c"].str.startswith("Airbus")      True/False for a prefix
