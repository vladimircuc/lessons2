Review

OOP and copying: what to remember

Three things carry most of the points here: inheritance (who can use which method), the dunder methods (and what happens when one is missing), and copying (alias vs shallow vs deep).

// inheritance

A child class gets all of the parent's methods for free, and can add its own. The parent does not get the child's methods. So a child object can call both, a parent object can only call the parent's.

inherit.py

// the dunder methods

These special methods let your objects behave like built-ins. The ones on this exam:

  • __init__(self, ...) runs when you build the object, sets attributes.
  • __eq__(self, other) defines ==, usually comparing attributes with and.
  • __lt__(self, other) defines <, used by sorting.
  • __repr__ / __str__ define how the object prints. If neither exists, printing gives the default <Class object at 0x...>, Python does not error, it just shows that.
dunders.py

// a method that changes the object

A regular method takes self, changes an attribute, and often has a condition. Watch for the ones that only act "if" something, and print a message otherwise.

method.py

// copying: alias vs shallow vs deep

This is the copying question the exam loves. Three levels:

  • Alias (b = a): same object, two names. Changing one changes the other.
  • Shallow copy (copy.copy(a)): a new outer object, but the inner objects (nested lists) are still shared.
  • Deep copy (copy.deepcopy(a)): a fully independent clone, all the way down. Nothing is shared.
copying.py

The trick they test

With a deep copy, changing the copy's nested list never touches the original. With a shallow copy or an alias, it does. Read carefully which copy was used and whether the change is to the outer object or a nested one.

Practice · OOP

OOP and copying questions

Multiple choice first, then the coding. Same shapes as the exam.

MC 1 multiple choice

What is true about these classes?
class Animal: defines eat(). class Dog(Animal): defines bark().

Correct: a Dog can use both. Dog inherits eat from Animal and defines its own bark. The last option is wrong because an Animal object cannot use bark, methods flow parent to child, not the other way.

MC 2 multiple choice

Given combo = {'bread': 'bun', 'main': ['beef', 'onion']} then new = copy.deepcopy(combo) and new['main'].append('cheese'). Which is true?

Correct: new is completely independent. deepcopy clones everything including the nested list, so appending to new['main'] leaves combo untouched. The last option describes a shallow copy, not a deep one.

MC 3 multiple choice

Maya makes a Coin class with only an __init__ method. What happens when she prints a Coin object?

Correct: Python prints a default. With no __str__ or __repr__, Python falls back to its built-in representation like <Coin object at 0x...>. It does not raise an error.

NumPy: what to remember

NumPy on this exam is mostly tracing, reading array code and predicting the output. The moves that show up: making arrays, boolean masking, np.where, and aggregates with an axis.

// making arrays and np.where

np.arange(start, stop, step) stops before stop. np.where(condition, a, b) gives a where the condition is true, b where it is false, element by element.

arange_where.py

// boolean masking

arr[condition] keeps only the elements where the condition is true. Combine conditions with & (and) and | (or), and wrap each condition in parentheses. Use ~ for not.

masking.py

Parentheses are not optional

Writing nums > 6 & nums < 15 without parentheses raises an error. Always (nums > 6) & (nums < 15).

// aggregates with an axis

On a 2D array, axis=1 works across each row (one result per row), and axis=0 works down each column. Without an axis you get one number for the whole array.

axis.py

remember for tracing

.mean() gives the average, np.sort(arr) returns the values smallest to largest, and elementwise math like arr * 2 or a / b applies to every element.

NumPy questions

MC 1 multiple choice

Which statement is TRUE about NumPy arrays and Pandas dataframes?

Correct. Arrays are tuned for fast numeric computation, dataframes hold labeled, mixed-type tabular data. Arrays are mutable, and a dataframe is 2D, so the other options are false.

MC 2 multiple choice

Which is a real advantage of a NumPy array over a plain Python list?

Correct. Arrays store one type in a compact block, which uses less memory and makes math much faster. They do not mix types, encrypt, or resize freely, those are the distractors.

Coding 3 coding · one line

Each row is [id, score, accident]. Label each student: 'Passed' if score is at least 600 and no accident, else 'Repeat' if no accident or score at least 300, else 'Try Again!'. Expected: ['Passed' 'Repeat' 'Passed' 'Repeat' 'Try Again!'].

try_it.py
what you write on the exam
np.where((arr[:, 1] >= 600) & (arr[:, 2] == 0), 'Passed',
         np.where((arr[:, 2] == 0) | (arr[:, 1] >= 300), 'Repeat', 'Try Again!'))

Two np.where calls nested: the outer handles Passed, the inner (the "else") handles Repeat vs Try Again. The array is all ints, so False becomes 0, and "no accident" is arr[:, 2] == 0. You can write it on one line, it is split here to read.

Pandas: what to remember

Pandas is the biggest section and it is almost all coding on a given DataFrame. The moves: selecting rows, grouping, making a column, and a few string and math helpers. Plus one trap that shows up in tracing.

// selecting rows

df[df['col'] condition] keeps matching rows. Combine conditions with & and |, each in parentheses. Pick a column off the result with ['name'].

select.py

// grouping, new columns, helpers

df.groupby('col')['other'].max() (or .mean(), .count()) gives one result per group. df['new'] = ... adds a column. Handy extras: .mean(), .isin([...]), and ['col'].str.replace(a, b).

group_col.py

// the inplace trap

Most Pandas operations return a new DataFrame and leave the original alone. If you do not reassign the result or pass inplace=True, nothing sticks. This is a favorite tracing gotcha.

inplace_trap.py

Read for this

If a line is df.something(...) with no df = in front and no inplace=True, it does nothing to df. Great for "what is the size after all this" questions where the answer is just the original.

Visualization with plotly express

This exam uses plotly.express as px, not matplotlib. You give it a DataFrame and the columns to use, it hands back a fig, and fig.show() displays it. These cells do not run here (plotly needs a browser renderer), so read them and picture the output.

// the common charts

import plotly.express as px

px.line(df, x="Name", y="Age")        line chart: x across, y up
px.bar(df, x="Name", y="Age")         bar chart
px.scatter(df, x="Height", y="Age")   scatter of two columns
px.histogram(df, x="Age", nbins=10)   histogram, 10 bins

fig = px.line(df, x="Name", y="Age")
fig.show()                             display it

Extras the exam asks for: color="Name" colors and splits by a column and adds a legend, nbins=10 sets the number of histogram bins, and title="..." sets the title.

// what px.line looks like

With x="Name" and y="Age", plotly puts each name along the x-axis and its age up the y-axis, then connects the points with a line, in the order the rows appear. Axes are labeled with the column names automatically.
