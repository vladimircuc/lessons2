The protocol

HTTP, the language of the web

HTTP stands for HyperText Transfer Protocol. It is just the set of rules browsers and servers use to talk. You ask for something (a request), and the server answers (a response). When you type a URL into a browser, that is an HTTP request under the hood asking the server to send back the page.

A get request asks a server to hand you data. Along with the data, the server sends a status code, a three digit number that tells you how the request went.

Status codes

The four families

You do not memorize hundreds of codes. You memorize four families by their first digit, plus a few specific ones.

FamilyMeaningPlain English
2xxSuccessIt worked.
3xxRedirectionIt moved, look elsewhere.
4xxClient errorYour request was wrong.
5xxServer errorThe server broke, not you.

The specific codes worth knowing

CodeMeans
200 OKRequest succeeded
301 Moved PermanentlyThe resource lives at a new URL now
401 UnauthorizedYou need to log in
403 ForbiddenYou are not allowed in
404 Not FoundNo such page
500 Internal Server ErrorSomething failed on the server
503 Service UnavailableServer is overloaded or down for now

Memory hook

4 is for "your fault" (you asked for the wrong thing, like a typo'd URL giving a 404). 5 is for "their fault" (the server fell over). 2 means smooth sailing.

Bots

Bots and robots.txt

A bot is a program that visits websites automatically instead of a human clicking around. A robots.txt file is a little note some sites post saying which parts a bot is allowed to visit. Google runs a bot called Googlebot to read the web and feed its search engine. You will use this idea again when we get to web scraping in Lecture 9.

Quick check

Test yourself

A request to a page that does not exist comes back with which code?

  • A200
  • B301
  • C404
  • D500

C, 404 Not Found. The page is missing, and a missing resource is a 4xx client error. A 500 would mean the server itself crashed, which is a different situation.

The server is temporarily overloaded and cannot respond. Which family does that code fall in?

  • A2xx
  • B3xx
  • C4xx
  • D5xx

D, 5xx. Anything that goes wrong on the server's side is a 5xx (this exact case is 503). It is not your request that is wrong, so it is not a 4xx.

The module

requests in four lines

The requests module is how Python sends HTTP requests. Almost everything you do starts with these four lines:

LineWhat it does
import requestsBrings in the module
r = requests.get(url)Sends a get request, hands back a response object
r.status_codeThe status code, as a number (an attribute, no parentheses)
r.textThe raw body of the response, always a string (an attribute)
r.json()The body converted into Python, only if it was JSON (a method, has parentheses)

What an API is

      
APIs hand you clean data

      
An API (Application Programming Interface) is a data endpoint built for programs, not for human eyes. Instead of a pretty web page, it returns structured data, almost always JSON. That is exactly what you want, because JSON drops straight into Python as lists and dictionaries.

      
Why reach for an API instead of scraping a web page? It is structured, it is stable, and it was built for this purpose. The tradeoff is that many APIs make you sign up for an API key so they know who is using their data. The Ghibli API is friendly and needs no key, which is why we use it here.

The classic mistake

      
The decoder error

      
r.json() only works when the body is actually JSON. If you call it on a normal web page, which is HTML, Python cannot parse it and throws an error. Run this and read the error, it is a good one to recognize.

      

        
decoder_error.py

        
        

      

      
Why it happens
The request itself succeeded (status 200), and r.text happily gives you the HTML as a string. The error only appears at r.json(), because there is no JSON to convert. .json() is not magic, it only works on JSON bodies.

    


    

      
Three facts that trip people up

      
Keep these straight

      
    
        
  • print(r) shows <Response [200]>, not the data. It is the object label with the status code inside.
    • 
        
  • r.text is always a string. Even if the server sent numbers or JSON, .text hands you raw text. Use .json() to turn JSON text into real Python.
    • 
        
  • An HTML page starts with < and usually ends with >. So for an HTML page, r.text[0] is "<" (the start of <!DOCTYPE> or <html>) and r.text[-1] is ">" (the end of </html>). The exam asks this directly.
    • 
      

      
Looking ahead
When a site has no API and you have to read the page's HTML yourself, that is web scraping, and you do it politely (check robots.txt, do not hammer the server). That is Lecture 9.

Which statement about APIs is FALSE?

C is the false one. APIs are about talking to remote services over the web, not about local files on your disk. The other three all describe what an API genuinely does.

      


      

        
What is the difference between using an API and web scraping?

        
    
          
  • AAn API edits a site, scraping edits an interface.
    • 
          
  • BAn API stores data, scraping stores interface state.
    • 
          
  • CAn API is a built endpoint for data, scraping reads the raw HTML.
    • 
          
  • DAn API reads raw HTML, scraping is a built endpoint.
    • 
        

        
        
C. An API is set up by developers to serve clean data through a defined endpoint. Web scraping means there is no such endpoint, so you pull the data straight out of the page's HTML yourself. D has the two reversed.

For a valid HTML web page, what is requests.get(url).text[0]?

The four lines:

        
import requests
r = requests.get(url)     # send the request
r.status_code             # the number (attribute)
r.text                    # the body as a STRING (attribute)
r.json()                  # body as Python, JSON only (method)

        
Status codes: 2xx success, 3xx redirect, 4xx your fault, 5xx server fault. Know 200, 403, 404, 500.

        
The API pattern (response to usable data):

        
data = requests.get(url).json()   # list of dicts, usually
data[0]["title"]                  # reach into one record

        
Facts the exam tests directly:
