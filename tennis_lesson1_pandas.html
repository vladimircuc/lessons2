What is df? After pd.read_csv(...), the variable df holds the whole table, called a DataFrame: rows are matches, columns are fields like surface or winner_name. Everything from here is asking df questions.

Look at the data before you touch it

The habit that separates people who are good with data from people who guess: always look first. Know how big the table is, what the columns are, and what the values look like.

Cell · size and column names

print(df.shape)            # (rows, columns)
print(df.columns.tolist()) # list of all 49 column names

Columns you will use a lot: surface, tourney_name, round, winner_name, loser_name, winner_rank, loser_rank, w_ace (winner aces), l_ace (loser aces), minutes.

Cell · first and last rows

df.head()      # first 5 rows, as a table
# df.head(10)  # first 10
# df.tail(3)   # last 3

In a notebook, df.head() as the last line of a cell prints a clean table, no print() needed.

Cell · column types and missing values

df.info()

Each column, how many non-empty values it has, and its type. Notice surface has a few blanks and winner_seed has many (unseeded players have no seed).

Cell · quick numeric summary

df.describe()

For every number column: count, mean, min, max, quartiles. Fast sanity check on ranges like minutes and w_ace.

Selecting columns and rows

Before filtering, you need to grab specific columns or specific rows.

Cell · one column, and several columns

df["winner_name"]                                   # a single column
df[["winner_name", "loser_name", "surface", "score"]]  # several columns (double brackets)

One name returns that column. A list of names (the inner [ ]) returns several columns as a smaller table.

Cell · rows by position with iloc

df.iloc[0]       # the very first row (all columns)
df.iloc[0:5]     # the first 5 rows

iloc means "integer location", so it selects rows by number.

Mental model. df["col"] picks columns. df.iloc[...] picks rows by position. The next tab picks rows by a condition, which is where the real power is.

Filtering rows with boolean masks

The most important skill in the lesson. A "mask" is a column of True/False, one per row. Put it inside df[ ... ] and pandas keeps only the True rows.

How it works, in two steps. A comparison on a column makes a True/False column:
df["surface"] == "Grass" → True for grass matches, False otherwise.
Feed it back in: df[ df["surface"] == "Grass" ] keeps the True rows. That is boolean indexing.

Cell · simple filters

df[df["surface"] == "Grass"]     # all grass-court matches
df[df["w_ace"] > 20]             # matches where the winner hit more than 20 aces
df[df["winner_name"] == "Carlos Alcaraz"]   # every match Alcaraz won

Cell · combining conditions

# AND uses &, OR uses |, each condition gets its own parentheses
df[(df["surface"] == "Grass") & (df["w_ace"] > 20)]     # grass AND 20+ aces
df[(df["surface"] == "Clay") | (df["surface"] == "Grass")]  # clay OR grass
Two rules people always trip on: use & and | (not the words and/or), and wrap each condition in parentheses. Miss either and you get an error.

Cell · isin and not

df[df["surface"].isin(["Clay", "Grass"])]   # surface is one of these (cleaner than many ORs)
df[~(df["surface"] == "Hard")]              # ~ flips it: everything NOT hard

Cell · counting how many match a condition

(df["surface"] == "Grass").sum()   # a mask is True/False; summing counts the Trues
len(df[df["w_ace"] > 20])          # or: filter, then count the rows

Both return a number. Summing a mask is the quick way to answer "how many matches were ...".

Sorting

Order the table by a column to find the biggest, smallest, longest, and so on.

Cell · sort by one column

# longest matches first
df.sort_values("minutes", ascending=False).head(10)

# show only the columns we care about
df.sort_values("w_ace", ascending=False)[["winner_name", "w_ace", "tourney_name"]].head(10)

ascending=False means largest first. Leave it out (or set True) for smallest first.

Cell · sort by two columns

# group surfaces together, and within each, longest match first
df.sort_values(["surface", "minutes"], ascending=[True, False]).head()

Cell · shortcut for top and bottom N

df.nlargest(10, "w_ace")[["winner_name", "w_ace"]]        # 10 biggest ace counts
df.nsmallest(5, "minutes")[["winner_name", "loser_name", "minutes"]]  # 5 shortest matches

Counting categories with value_counts

value_counts() tallies how many times each value appears in a column. Great for "how many matches per surface" or "who won the most".

Cell · basic counts

df["surface"].value_counts()        # matches on each surface
df["tourney_level"].value_counts()  # G = Grand Slam, M = Masters, A = ATP event, D = Davis Cup

Each value with its count, biggest first. Surfaces are Hard, Clay, Grass.

Cell · most wins, and proportions

df["winner_name"].value_counts().head(10)   # players with the most wins in 2023
df["surface"].value_counts(normalize=True)  # same, but as fractions of the total

Cell · how many distinct values

df["winner_ioc"].nunique()   # how many different countries produced a winner
df["surface"].unique()       # the list of distinct surfaces

Grouping and aggregating with groupby

The most powerful tool this week. groupby splits the table into groups (by surface, say), then computes something for each group (average aces, say). This turns raw matches into insight.

The pattern: df.groupby("WHAT_TO_GROUP_BY")["WHICH_COLUMN"].WHAT_TO_COMPUTE()
Read left to right: split by that, look at that column, compute that.

Cell · one number per group

df.groupby("surface")["w_ace"].mean()     # average winner aces per surface
df.groupby("surface")["minutes"].mean()   # average match length per surface

One row per surface. This is where you would expect grass to show more aces than clay, and you can check it in one line.

Cell · several stats at once with agg

df.groupby("surface")["w_ace"].agg(["mean", "max", "min", "count"])

agg takes a list of things to compute, giving a tidy table with a column for each.

Cell · group by one thing, summarize two columns

df.groupby("surface")[["w_ace", "w_df"]].mean()   # avg aces and double faults per surface

Cell · combine groupby with sorting

# total aces each winner served across the season, most first
df.groupby("winner_name")["w_ace"].sum().sort_values(ascending=False).head(10)

Basic statistics

Single-number summaries of a column.

Cell · the common ones

df["w_ace"].mean()     # average
df["w_ace"].median()   # middle value
df["w_ace"].std()      # spread (standard deviation)
df["minutes"].max()    # longest match in minutes
df["minutes"].min()    # shortest
df["w_ace"].sum()      # total

Cell · summarize several columns together

df[["w_ace", "l_ace", "minutes"]].describe()

Cell · a first taste of correlation

# does hitting more aces go with more double faults? (value from -1 to 1)
df["w_ace"].corr(df["w_df"])

Correlation measures how two numbers move together. Near 0 means little relationship. We lean on this heavily when picking features for the predictor.

Creating new columns

Build new columns from existing ones. This is the seed of "feature engineering", which is most of the work in the AI project.

Cell · make two useful columns

df["total_aces"] = df["w_ace"] + df["l_ace"]         # aces by both players combined
df["rank_diff"]  = df["loser_rank"] - df["winner_rank"]  # ranking gap (positive = favorite won)

df[["winner_name", "loser_name", "total_aces", "rank_diff"]].head()

A positive rank_diff means the higher-ranked player (lower rank number) won. That single idea previews how we will frame the prediction problem.

Practice

Different from the examples and a bit harder on purpose, but everything you need is in the other tabs. Each in its own cell. Try first, then open the solution.

1.Show every Grand Slam final from 2023. warm-up

Hint: Grand Slams are tourney_level == "G", a final is round == "F". Show tournament, winner, loser, score.

Solution
df[(df["tourney_level"] == "G") & (df["round"] == "F")][
    ["tourney_name", "winner_name", "loser_name", "score"]
]

Two conditions joined with &, then a list of columns to display.

2.Which player won the most matches on clay? warm-up

Hint: filter to clay first, then count winners.

Solution
df[df["surface"] == "Clay"]["winner_name"].value_counts().head(1)

Filter to clay, take winner_name, tally it, keep the top one.

3.Average match length per surface, sorted longest to shortest. step up

Solution
df.groupby("surface")["minutes"].mean().sort_values(ascending=False)

Groupby gives the average per surface, then sort the result.

4.Top 5 matches by total aces (winner plus loser). Show both players and the tournament. step up

Hint: you may need a new column first.

Solution
df["total_aces"] = df["w_ace"] + df["l_ace"]
df.nlargest(5, "total_aces")[["winner_name", "loser_name", "total_aces", "tourney_name"]]

Build the column, then nlargest pulls the top 5 by it.

5.In how many matches did the higher-ranked player win? (winner_rank smaller than loser_rank) step up

Solution
(df["winner_rank"] < df["loser_rank"]).sum()

The comparison makes a True/False column; summing it counts how often the favorite won. This is basically the accuracy of the simplest predictor, "the higher-ranked player wins", which we will beat later.

6.For each tournament level, show the average winner aces and how many matches there were. challenge

Solution
df.groupby("tourney_level")["w_ace"].agg(["mean", "count"])

agg with two functions gives both numbers side by side, one row per level.

7.Find every match longer than 3 hours played on grass, and show how many there were. challenge

Hint: 3 hours is 180 minutes; combine two conditions, then count.

Solution
long_grass = df[(df["minutes"] > 180) & (df["surface"] == "Grass")]
long_grass[["winner_name", "loser_name", "minutes", "tourney_name"]]   # see them
len(long_grass)                                                        # count them

Save the filtered table to a variable so you can both view it and count it.
