What this is

A database is a set of tables

A table is a grid: rows are records (one park, one amenity) and columns are fields (its name, its acreage). SQL is the language you use to ask a table questions and get rows back. Instead of opening CSV files and looping in Python, you write one statement and the database does the work.

Tools for Q2

LIKE, AND, and DESC

LIKE matches a text pattern instead of an exact value. The % wildcard stands for any run of characters, so '%river%' means "has the word river anywhere in it." LIKE is case-insensitive, so '%river%' and '%RIVER%' match the same rows.

like.sql

AND requires both conditions, OR requires either. Q2 needs both an island description and a size, so it uses AND.

Second homework query

Homework Q2

Return the name and acreage of each park whose description contains "island" (case does not matter) and that has more than 50,000 acres, sorted by acreage from largest to smallest.

Homework Q2 homework  no_man_is_an_island

your_query.sql
goes in your HW08.py, question 2
query = """
    SELECT name, area_acres
    FROM parks
    WHERE description LIKE '%island%' AND area_acres > 50000
    ORDER BY area_acres DESC;
"""

LIKE with %island% on both sides finds the word anywhere in the description, AND adds the size test, and DESC puts the biggest parks first. This one matches the expected output exactly on the sample.

Tools for Q3

LIMIT: the top few

LIMIT n keeps only the first n rows of the result. On its own it just cuts the list short, but paired with ORDER BY ... DESC it becomes "the top n," which is exactly what Q3 wants.

limit.sql

order matters

ORDER BY comes before LIMIT. The database sorts everything first, then hands you the top slice. Flip the sort to ASC and the same LIMIT would give you the bottom few instead.

Third homework query

Homework Q3

Return the name and visitor spending for the four parks with the highest number of visits.

query = """
    SELECT park_name, visitor_spending
    FROM park_gdp

Note you sort by visits but display visitor_spending, the column you order by does not have to be one you select. Matches the expected output exactly.

Tools for Q4

COUNT, GROUP BY, HAVING

An aggregate function squashes many rows into one number. COUNT(*) counts rows.

count.sql

GROUP BY splits the rows into groups that share a value, then the aggregate runs once per group. Grouping the amenities by name and counting gives how many times each amenity appears.

group_by.sql

HAVING filters the groups after grouping. It looks like WHERE, but WHERE filters individual rows before grouping, while HAVING filters the grouped results and can test the aggregate itself.

having.sql

WHERE vs HAVING

Rule of thumb: filter raw rows with WHERE, filter counts and other aggregates with HAVING. You cannot put COUNT(*) > 2 in a WHERE, it has to be HAVING.

Fourth homework query

Homework Q4

Return the name and count of the amenities that appear more than 375 times, in descending order of the count.

Homework Q4 homework  frequent_features

your_query.sql
goes in your HW08.py, question 4
query = """
    SELECT name, COUNT(*)
    FROM amenities
    GROUP BY name
    HAVING COUNT(*) > 375
    ORDER BY COUNT(*) DESC;
"""

Group by name, count each group, keep only the big ones with HAVING, then order by the count. On this small sample nothing reaches 375, so it returns 0 rows, but on your full database it gives the 5 amenities in the expected output. Try the teaching example above with a count over 2 to see the same query shape return rows.

Lecture 17 cheat sheet

The query skeleton

SELECT columns
FROM table
WHERE row_condition
GROUP BY column
HAVING group_condition
ORDER BY column ASC | DESC
LIMIT number;

Picking and filtering

SELECT name, area_acres FROM parks;      chosen columns
SELECT * FROM parks;                     all columns
WHERE area_acres > 300000                number compare: = < > <= >= <>
WHERE state = 'Utah'                     text in single quotes
WHERE a > 5 AND b = 'x'                  both must hold
WHERE a = 'x' OR a = 'y'                 either holds

Pattern matching

WHERE description LIKE '%island%'    contains the word (case-insensitive)
WHERE name LIKE 'A%'                 starts with A
%  matches any run of characters

Sorting and limiting

ORDER BY area_acres          ascending (default)
ORDER BY area_acres DESC     descending
ORDER BY visits DESC LIMIT 4 the top 4

Counting in groups

COUNT(*)                                  count rows
GROUP BY name                             one group per distinct name
HAVING COUNT(*) > 375                     filter groups by the count
WHERE filters rows, HAVING filters groups
