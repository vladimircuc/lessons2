Why this exists

Why NumPy instead of a list

A Python list holds anything and grows freely, which is flexible but slow. A NumPy array is the opposite, and that is the point:

  • One type only. Every element is the same kind, usually all ints or all floats. That uniformity is what makes it fast.
  • Fixed size. The memory is laid out tightly when you make it, so the computer can tear through it.
  • Operate on it all at once. Write array + 7 and every element gets 7 added in one shot, no loop. This is vectorizing, and it is faster and easier to read.

Filling arrays for you

      
arange, linspace, and the fillers

      
np.arange(start, stop, step)
"count from start up to but not including stop, jumping by step." Like range, but the step can be a decimal and you get an array.

      
np.linspace(start, stop, num)
"give me num values evenly spread from start to stop." The stop is included by default, and the result is floats.

      

        
fillers.py

        
        

      

      
np.arange(9).reshape(3, 3) builds a flat range then folds it into a shape, and np.random.random((2, 4)) fills a block with random decimals from 0 to 1.

Filtering

      
Masking

      
A comparison like a > 3 does not give one True or False, it gives a whole array of them, one per element. That boolean array is the mask. Feed it back in and you keep only the elements where it was True.

      

        
masking.py

        
        

      

      
The one thing to remember
Masking always hands back a 1D array, whatever the original shape, because the True cells are usually scattered.

      
Pick a condition and watch the mask light up. Green cells pass, the strip on the right is the 1D result.

Two conditions at once

      
Combining masks

      
Combine masks with bitwise operators, not the words and or or: & means and, | means or, ~ means not. Wrap each condition in its own parentheses.

      

        
combine.py

        
        

      

      
The parentheses are not optional
Write (a1 % 2 == 0) & (a1 > 3), not a1 % 2 == 0 & a1 > 3. Without them the & runs first and errors.

Lecture 10 cheat sheet

      

        
Creating arrays

        
np.array([1,2,3])             from a list (or list of lists for 2D)
np.arange(start, stop, step)  range, stop NOT included, step can be decimal
np.linspace(start, stop, num) num evenly spaced values, stop IS included
np.zeros((r, c))              block of zeros (floats)
np.ones((r, c), dtype=int)    block of ones
np.random.random((r, c))      random decimals in [0, 1)
arr.reshape((r, c))           same values, new shape
.shape  .dtype                the size tuple, and the element type

        
Indexing and slicing

        
a[2]         one element (negatives count from the end)
a[1, 2]      2D: row 1, column 2
a[0:2, 1:3]  rows 0-1, columns 1-2  (stop not included)
a[:, 0]      every row, column 0  (a whole column)
a[2, ::2]    row 2, every other column

        
Vectorized math

        
a + 7        a scalar broadcasts to every element
a + b        same-shape arrays combine cell by cell
a * b        cell by cell, NOT matrix mult
np.matmul(a, b)   true matrix multiplication
a.flatten()  squash any array to 1D
np.round(a, 1)    round to 1 decimal place

        
Masking

        
a > 3              a mask: an array of True/False
a[a > 3]           keep only the True elements (ALWAYS 1D)
a[a % 2 == 0]      only the evens
(a > 5) & (a < 15) combine with & | ~  and wrap each piece in ()
