Lecture 10: Intro to NumPy

NumPy is the array library that everything else in data work sits on. The idea is simple: one big block of numbers, all the same type, that you operate on all at once instead of looping. The diagrams here are interactive, so drag the controls and watch the arrays change.

Why this exists

Why NumPy instead of a list

A Python list can hold anything and grow forever, which is flexible but slow. A NumPy array is the opposite, and that is the whole point:

  • One type only. Every element is the same kind, usually all ints or all floats. That uniformity is what makes it fast.
  • Fixed size. The block of memory is set when you make it, so the computer can lay it out tightly and tear through it.
  • Operate on it all at once. Instead of a loop, you write array + 7 and every element gets 7 added in one shot. This is called vectorizing, and it is both faster and easier to read.

You always bring it in the same way, and it is conventionally shortened to np:

Building blocks

      
Making an array, and its shape

      
The most direct way is np.array, handing it a list or a list of lists. A flat list gives a 1D array, a list of lists gives a 2D array, a grid of rows and columns.

Filling arrays for you

      
arange, linspace, and the fillers

      
You rarely type out every number. These build arrays for you.

      
np.arange(start, stop, step)
"count from start up to but not including stop, jumping by step." Like Python's range, but the step can be a decimal, and you get an array back.

      
np.linspace(start, stop, num)
"give me num values evenly spread from start to stop." By default it includes the stop value, and the result is floats.

      

        
fillers.py

        
        

      

      
np.arange(9).reshape(3, 3) is a common combo: build a flat range, then fold it into a shape. And np.random.random((2, 4)) fills a block with random decimals between 0 and 1.

Reaching in

      
Indexing

      
A 1D array indexes just like a list, including negative indices from the end. A 2D array uses a single pair of brackets with a comma, a[row, column], which is cleaner than the a[row][column] you might expect.

      

        
indexing.py

        
        

      

      

        a[1, 2] picks one cell

Taking a chunk

      
Slicing

      
Slicing uses start:stop, with the stop not included, exactly like lists. For 2D you slice both dimensions at once, separated by the comma.

      
a[0:2, 1:3]
"rows 0 up to 2, and columns 1 up to 3." The part before the comma picks rows, the part after picks columns.

Array with array

      
Elementwise, reshape, and matmul

      
Two arrays of the same shape combine cell by cell. Reshaping rearranges the same values into a new shape, and .flatten() squashes any array down to 1D.

      

        
array_ops.py

        
        

      

      
Watch out
The star * between two arrays multiplies cell by cell, it is not matrix multiplication. For real matrix multiplication use np.matmul(a, b) or the @ operator.

Filtering

      
Masking

      
Masking is how you filter an array by a condition. A comparison like a > 3 does not give one True or False, it gives a whole array of True and False, one per element. That boolean array is the mask. Feed it back into the array and you get only the elements where the mask was True.

      

        
masking.py

        
        

      

      
The one thing to remember
Masking always hands back a 1D array, no matter the shape of the original. It cannot keep a rectangle shape, because in general the True cells are scattered.

      
Pick a condition and watch the mask light up. Green cells pass, and the strip on the right is the 1D array you get back.

Two conditions at once

      
Combining masks

      
To use two conditions together you combine the masks with bitwise operators, not the words and or or: & means and, | means or, ~ means not. Each condition must be wrapped in its own parentheses, because these operators bind tighter than the comparisons.

      

        
combine.py

        
        

      

      
The parentheses are not optional
Write (a1 % 2 == 0) & (a1 > 3), not a1 % 2 == 0 & a1 > 3. Without the parentheses the & tries to run first and you get an error. Tick the second condition in the explorer above to see two masks combine.

Lecture 10 cheat sheet

      

        
Creating arrays

        
np.array([1,2,3])             from a list (or list of lists for 2D)
np.arange(start, stop, step)  range, stop NOT included, step can be decimal
np.linspace(start, stop, num) num evenly spaced values, stop IS included
np.zeros((r, c))              block of zeros (floats)
np.ones((r, c), dtype=int)    block of ones
np.random.random((r, c))      random decimals in [0, 1)
arr.reshape((r, c))           same values, new shape (returns a new array)
.shape  .dtype                the size tuple, and the one element type

        
Indexing and slicing

        
a[2]        one element (negative counts from the end)
a[1, 2]     2D: row 1, column 2
a[0:2, 1:3] rows 0-1, columns 1-2  (stop not included)
a[:, 0]     every row, column 0  (a whole column)
a[-1]       the last row

        
Vectorized math

        
a + 7        a scalar broadcasts to every element
a + b        same-shape arrays combine cell by cell
a * b        cell by cell, NOT matrix mult
np.matmul(a, b)   true matrix multiplication
a.flatten()  squash any array to 1D

        
Masking

        
a > 3            a mask: an array of True/False
a[a > 3]         keep only the True elements (ALWAYS returns 1D)
a[a % 2 == 0]    only the evens
(a > 3) & (a < 8)  combine with & | ~  and wrap each piece in ()

        
Worth memorizing

        
- masking always returns a 1D array
- a slice is a view into the same data, not a copy
- check .shape whenever something looks wrong
- use & | ~ for masks (not and/or/not), with parentheses
