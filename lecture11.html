Boiling it down

Aggregates

An aggregate squeezes a whole array into a summary: .sum(), .mean(), .max(), .min(). With no argument they use every element. The interesting part is the axis.

axis=0 goes down the columns, axis=1 goes across the rows

So a.sum(axis=0) gives one number per column, and a.sum(axis=1) gives one number per row.

aggregates.py

      

      
This widget makes the axis concrete. The amber cells below are the per-column result (axis 0), the amber cells on the right are the per-row result (axis 1), and the corner is the whole-array value.

function
          
            on a = [[1, 2, 3], [4, 5, 6]]
        

        

        

        
bottom row = axis 0 (down columns)  ·  right column = axis 1 (across rows)  ·  corner = whole array

Replace by condition

      
np.where

      
np.where builds a new array by choosing, for each element, between two options based on a condition.

      
np.where(condition, value_if_true, value_if_false)
"wherever the condition is True use the first value, otherwise use the second." The two values can be single values or whole arrays.

For three or more buckets you nest a where inside the false slot, like an if then elif then else.

Joining arrays

      
np.concatenate

      
np.concatenate glues arrays together. You pass the arrays as a single tuple, and an optional axis says which direction to join in.

      

        
concatenate.py

        
        

      

      
Note the double parentheses
The arrays go inside one tuple: np.concatenate((a, b)), not np.concatenate(a, b). axis=0 joins top to bottom (more rows), axis=1 joins side to side (more columns).

Missing data

      
nan and np.isnan

      
np.nan is NumPy's "not a number," the placeholder for a missing value. It has one trap that drives the whole topic: nan is not equal to anything, not even itself. So x == np.nan never works. You detect it with np.isnan.

      

        
isnan.py

        
        

      

      
Random integers
You will also want np.random.randint(low, high), which gives a random whole number from low up to but not including high. Calling np.random.seed(n) first makes the result repeatable.

Your source of truth

      
Lecture 11 cheat sheet

      

        
Aggregates

        
a.sum()  a.mean()  a.max()  a.min()   the whole array
a.sum(axis=0)    one value per column (down)
a.sum(axis=1)    one value per row (across)
a.sum(axis=0).max()   chain them, the first returns an array

        
Return-new vs in-place

        
NEW array : np.sort(a), a.reshape(), a.copy(), a.transpose(), a.astype()
IN PLACE  : a.sort(), a.resize(), a.fill()   (these return None)
a.astype(int)  converts type; float -> int TRUNCATES, no rounding

        
np.where

        
np.where(cond, if_true, if_false)
np.where(a >= 80, "A", np.where(a >= 60, "B", "F"))   nest for 3+ buckets
the true/false values can be single values or whole arrays

        
Concatenate

        
np.concatenate((a, b))          arrays go in ONE tuple
np.concatenate((a, b), axis=0)  top to bottom (more rows)
np.concatenate((a, b), axis=1)  side to side (more columns)
np.concatenate((a, b), axis=None)  flatten into one 1D array

        
Missing data and files

        
np.nan              the missing-value placeholder
np.nan == np.nan    is False, so never compare with ==
np.isnan(a)         mask of where the nans are
a[~np.isnan(a)]     the values that are NOT missing
np.savetxt(f, a, delimiter=",", fmt="%d")   write
np.loadtxt(f, delimiter=",", skiprows=1)    read (skip a header)
