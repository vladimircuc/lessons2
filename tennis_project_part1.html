Set up the project notebook

Create the file and lay out its skeleton first, so the whole project has a clean, professional structure from day one.

Why we scaffold it up front.

A good project notebook reads like a story: load, clean, build features, model, evaluate. If you just pile cells on week after week it turns into an unreadable mess that will not run top to bottom. So we create all the section headers now, even the ones for future weeks, and each week you fill in the right section. Keeping it tidy is itself a skill, and it is what makes the final project look impressive to a college reviewer.

Section 2: Clean the Data

Under ## 2. Clean the Data. Two parts: fix the date column, then handle missing values.

Task A · fix the date

Convert tourney_date from a plain number into a real date, then add year and month columns.

Solution
df["tourney_date"] = pd.to_datetime(df["tourney_date"], format="%Y%m%d")
df["year"]  = df["tourney_date"].dt.year
df["month"] = df["tourney_date"].dt.month

df[["tourney_date", "year", "month"]].head()

You should see

tourney_date now showing real dates like 2023-01-02, with matching year and month columns.

Task B · look at the gaps

Before fixing anything, see where the missing values are. Print the missing count per column, worst first.

Solution
df.isna().sum().sort_values(ascending=False).head(10)

You should see

The seed columns near the top (mostly empty, expected), then smaller gaps in the serve stats, winner_rank, loser_rank, and surface. Those last three are the ones we care about for features.

Task C · handle the gaps

Drop the rows missing the essentials we need (surface, winner_rank, loser_rank), then fill missing heights with the median so we keep those rows. Print the shape before and after.

Solution
print("before:", df.shape)

# drop rows missing something we cannot fake
df = df.dropna(subset=["surface", "winner_rank", "loser_rank"])

# fill minor gaps with the median height
df["winner_ht"] = df["winner_ht"].fillna(df["winner_ht"].median())
df["loser_ht"]  = df["loser_ht"].fillna(df["loser_ht"].median())

print("after: ", df.shape)

You should see

The row count drop from 2986 to somewhere around 2800 (the exact number depends on how many rows were missing a rank or surface), and no more gaps in the height columns.

Add a short markdown note above this section recording your choices, for example: "Dropped rows missing surface or either ranking, since those cannot be reliably guessed. Filled missing heights with the median." Writing down your cleaning decisions is exactly the kind of reasoning that stands out in a project.

Section 3: Build Features

Under ## 3. Build Features. Turn the raw columns into the difference features a model can learn from.

Your task

Build four pre-match difference features: rank_diff, points_diff, age_diff, and height_diff. Then show the first few rows of just those columns.

Solution
df["rank_diff"]   = df["loser_rank"] - df["winner_rank"]
df["points_diff"] = df["winner_rank_points"] - df["loser_rank_points"]
df["age_diff"]    = df["winner_age"] - df["loser_age"]
df["height_diff"] = df["winner_ht"]  - df["loser_ht"]

df[["rank_diff", "points_diff", "age_diff", "height_diff"]].head()

You should see

A small table with your four new columns. Positive rank_diff and points_diff mean the higher-ranked, higher-points player won.

Only pre-match columns. Every feature above comes from information known before the match starts. Do not build features from aces, double faults, minutes, or the score, those are post-match and would leak the result (see the Leakage tab in the lesson). This is the single most important rule of the whole project.

Bonus · the number to beat

As a sanity check, compute how often the higher-ranked player won. This is the "always pick the favorite" baseline our model has to beat later.

Solution
favorite_wins = (df["rank_diff"] > 0).sum()
known = df["rank_diff"].notna().sum()
print(favorite_wins / known)

You should see

A number around 0.65, meaning the favorite wins roughly two-thirds of the time. Write it down. When we train a real model, beating this number is how we know it actually learned something.

Add a markdown note under this section: "These features currently use the winner and loser columns, which encode the result. Next week we will restructure the data into a symmetric player A vs player B format with a proper 0/1 label so the model cannot cheat." That documents the next step and shows you understand the trap.
