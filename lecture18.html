What this is

Building on Lecture 17

Lecture 17 gave you SELECT, WHERE, LIKE, ORDER BY, GROUP BY, COUNT, and HAVING. This lecture adds three things the last four questions need: more aggregate functions with named output columns, subqueries (a query used inside another), and joins (pulling from more than one table at once).

parks

name, state, date_established, area_acres, description

park_gdp

park_name, visits, visitor_spending, economic_output

amenities

id, name, categories, parkCode

park_codes

code, name

how the tables connect

park_gdp.park_name matches parks.name. Amenities link to a park through amenities.parkCode equals park_codes.code, and park_codes.name matches parks.name. Those matching columns are what a join lines up on.

SUM, and naming a column with AS

You met COUNT in Lecture 17. SUM, AVG, MAX, and MIN work the same way, one number per group. SUM(area_acres) with GROUP BY state adds up the acreage in each state.

Fifth homework query

Homework Q5

Show each state and the total acres of its National Parks, named total_area. Sort by that total, largest first, and show only the top three states.

Homework Q5 homework  park_powerhouse

your_query.sql
goes in your HW08.py, question 5
query = """
    SELECT state, SUM(area_acres) AS total_area
    FROM parks
    GROUP BY state
    ORDER BY total_area DESC
    LIMIT 3;
"""

Group by state, sum the acres and name it, then sort by that name and cap at three. Matches the expected output (Alaska, California, Wyoming) on the full database.

AVG, aliases with spaces, and subqueries

AVG gives the average of a column per group. When an alias has a space in it, wrap it in double quotes, like AS "Average Acres".

Subqueries in HAVING

Q6 puts two subqueries in the HAVING, one for the overall average acreage, one for Texas's park count. Each runs once and gives a number the groups are tested against.

Sixth homework query

Homework Q6

Show the state, its average acre size as "Average Acres", and its park count as "Number of Parks". Keep only states whose average is below the average of the whole parks table AND that have more parks than Texas. Order by number of parks descending, and by state name for ties.

Homework Q6 homework  crazy_park_question

your_query.sql
goes in your HW08.py, question 6
query = """
    SELECT state, AVG(area_acres) AS "Average Acres", COUNT(*) AS "Number of Parks"
    FROM parks
    GROUP BY state
    HAVING AVG(area_acres) < (SELECT AVG(area_acres) FROM parks)
       AND COUNT(*) > (SELECT COUNT(*) FROM parks WHERE state = 'Texas')
    ORDER BY COUNT(*) DESC, state;
"""

Group by state, then HAVING filters those groups with two subqueries joined by AND. The order clause sorts by the count, then breaks ties alphabetically by state. This matches the full database's expected output exactly, all seven states in order.

Tools for Q7

Joining two tables

Amenities only store a parkCode like ACAD, not a readable park name. The name lives in park_codes. A JOIN lines up rows from the two tables where the codes match, so you can use columns from both.

join.sql

two things to notice

You write table.column so it is clear which table a column comes from. And ON says which columns must match, here amenities.parkCode = park_codes.code.

A park has many amenities, so joining repeats the park name once per amenity. DISTINCT collapses those repeats so each park appears once.

distinct.sql

Seventh homework query

Homework Q7

Return the distinct park names that have any bicycle-related amenity, in ascending order. An amenity is bicycle related if its name contains the word "Bicycle".

Homework Q7 homework  worrisome_wheels

your_query.sql
goes in your HW08.py, question 7
query = """
    SELECT DISTINCT park_codes.name
    FROM amenities
    JOIN park_codes ON amenities.parkCode = park_codes.code
    WHERE amenities.name LIKE '%Bicycle%'
    ORDER BY park_codes.name;
"""

The join turns each amenity's code into its park name, the LIKE keeps only bicycle amenities, and DISTINCT makes sure a park with several bicycle amenities is listed once. The expected output uses the uppercase names from park_codes, and the full database returns 45 of them.

Joining several tables and counting

Q8 pulls from three tables at once: the park's name and state from parks, its spending and visits from park_gdp, and a count of its amenities. You chain joins, one JOIN ... ON per table you add.

To count amenities per park, join amenities in and use COUNT with GROUP BY. Each park's group holds one row per amenity, so counting them gives the number of amenities.

Eighth homework query

Homework Q8

For each park with more than 2,000,000 visits, return the name, state, number of amenities, and visitor spending, sorted by visitor spending descending.

Homework Q8 homework  scalable_impact

your_query.sql
goes in your HW08.py, question 8
query = """
    SELECT parks.name, parks.state, COUNT(amenities.id) AS num_amenities, park_gdp.visitor_spending
    FROM parks
    JOIN park_gdp ON parks.name = park_gdp.park_name
    JOIN park_codes ON parks.name = park_codes.name
    JOIN amenities ON amenities.parkCode = park_codes.code
    WHERE park_gdp.visits > 2000000
    GROUP BY parks.name, parks.state, park_gdp.visitor_spending
    ORDER BY park_gdp.visitor_spending DESC;
"""

Three joins chain the four tables: park_gdp for spending and visits, park_codes to bridge the name to the code, amenities to count. GROUP BY collapses each park to one row so COUNT gives its amenity total, WHERE keeps the busy parks, and the order is by spending. This matches all 13 rows of the expected output on the full database.

Lecture 18 cheat sheet

Aggregates and names

SUM(col)  AVG(col)  MAX(col)  MIN(col)  COUNT(*)
SELECT state, SUM(area_acres) AS total_area ...   name a column
AS "Average Acres"                                quotes if the name has a space
ORDER BY total_area DESC                          you can sort by the alias

Subqueries

WHERE area_acres > (SELECT AVG(area_acres) FROM parks)
HAVING COUNT(*) > (SELECT COUNT(*) FROM parks WHERE state = 'Texas')
a query in parentheses that returns one value to compare against

Joining tables

SELECT a.name, pc.name
FROM amenities AS a
JOIN park_codes AS pc ON a.parkCode = pc.code;

/* chain more tables, one JOIN ... ON each */
FROM parks
JOIN park_gdp   ON parks.name = park_gdp.park_name
JOIN park_codes ON parks.name = park_codes.name
JOIN amenities  ON amenities.parkCode = park_codes.code

Join helpers

table.column          say which table a column is from
DISTINCT              one row per unique value (drop repeats)
COUNT(col) + GROUP BY count rows per group after joining

The full order of clauses

SELECT ... FROM ... JOIN ... ON ...
WHERE ... GROUP BY ... HAVING ...
ORDER BY ... LIMIT ...;
