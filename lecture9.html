Why we scrape

When there is no API

Back in Lecture 6 you saw that an API hands you clean, ready-to-use data. That is the easy case. But plenty of websites have no API at all. They only have the web page itself. Web scraping is what you do then: you download the page's HTML and dig the data out of it yourself.

The plan is two steps:

  1. Get the page's HTML as a string. You already know how, from Lecture 6: requests.get(url).text hands you the whole page as text.
  2. Parse that HTML with BeautifulSoup. Raw HTML is an awkward wall of text. BeautifulSoup turns it into something you can search, so you can ask for "all the table rows" or "the first link" without hunting by hand.

Scrape politely

Same manners as Lecture 6. Check the site's robots.txt to see what it allows, and do not hammer a server with rapid-fire requests.

What you are digging through

HTML in a few minutes

To pull data out of a page you need to recognize its shape. HTML is built out of tags.

A tag wraps a piece of content. <td>Everest</td> has an opening tag <td>, the content Everest, and a closing tag </td>. The three together are called an element.

Tags nest inside each other. A table row, <tr>, holds several cells, <td>. A whole table, <table>, holds several rows. So the page is a tree of tags inside tags.

Attributes are extra labels written inside the opening tag, as name="value" pairs. For example <td class="name"> gives that cell a class of "name", and <a href="https://..."> gives a link its web address. Attributes are how you tell similar tags apart.

Step one

Turning HTML into soup

You start by handing the HTML to BeautifulSoup. Two lines:

from bs4 import BeautifulSoup
soup = BeautifulSoup(html, "html.parser")

Import the tool, then build a soup object from your HTML. The second argument, "html.parser", names the engine that reads the HTML. It is built into Python, so it always works.

That word "soup" is just BeautifulSoup's name for the parsed page. Once you have it, you can search the page instead of scanning raw text. The html here is the page's HTML as a string. In a real program you would fetch it first with requests.get(url).text from Lecture 6, but on this page it is already loaded for you.

A first taste: soup.title.text reaches into the <title> tag and pulls out its text.

The two workhorses

      
find and find_all

      
Almost all scraping comes down to these two methods, and the difference is simple.

      

        
MethodReturns

        
soup.find("td")the FIRST matching tag, or None if there are none

        
soup.find_all("td")a LIST of every matching tag

      

      
It is the same one-versus-all idea you saw with search and findall in regex. Use find when you want one thing, find_all when you want all of them.

Searching by class
To find tags with a certain class, pass class_ with an underscore, like soup.find_all("td", class_="name"). The underscore is there because class on its own is a reserved word in Python, so BeautifulSoup uses class_ instead.

Getting the data out

      
Text and attributes

      
Once you have a tag, there are two things you usually want from it: the text inside it, and its attributes.

      
Text: .text (you may also see .get_text()) gives you the visible text inside a tag, with the surrounding tags stripped away.

      
Attributes: to read an attribute, treat the tag like a dictionary and look it up by name. tag["href"] gives a link's web address. Same square-bracket lookup you already use for dictionaries.

The everyday pattern

      
Scraping a whole table

      
Here is the move you will use again and again: grab all the rows, loop over them, and for each row grab its cells and read the ones you care about.

      
The recipe
rows = soup.find_all("tr"), then for each row, cells = row.find_all("td"), then read cells[0], cells[1], and so on.

      

        
scrape_table.py

        
        

      

      
Walk through it: find_all("tr") gives the three rows. Inside the loop, row.find_all("td") gives that one row's cells. cells[0] is the name cell and cells[2] is the country cell, so each loop prints a name and its country.

Setup:

        
from bs4 import BeautifulSoup
soup = BeautifulSoup(html, "html.parser")

        
Finding tags:

        
soup.find("td")              first match, or None
soup.find_all("td")          list of every match
soup.find_all("td", class_="name")   by class (note the _)

        
Reading a tag:

        
tag.text         the visible text inside it
tag["href"]      an attribute, looked up like a dict key

        
The table loop:

        
for row in soup.find_all("tr"):
    cells = row.find_all("td")
    name = cells[0].text
    # cell text is a string, use int() to compare numbers

        
API versus scraping (a favorite multiple choice): an API is a built endpoint that serves clean data, while scraping pulls data straight out of a page's HTML because there is no such endpoint.
