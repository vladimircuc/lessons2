The exam pulls from the whole course, so the practice here is split the same way. Each tab holds the question types that come from that lecture:

  • L3 One-liners, comprehensions, sorted, lambda, zip. The tracing and one-line coding sections.

  • L8 Regex 2, capture groups, greedy versus lazy, and sub.

Two kinds of question

Coding and tracing questions give you a real code box: write or predict, hit Run, and the answer prints out. Multiple choice questions are clickable, the option turns green if it is right and red if it is not. Either way, open Show solution for a short why. On the BeautifulSoup tab the first Run loads the library, so give it a few seconds.

Lecture 6

      
Requests and APIs

      
These come up as multiple choice on the exam, since you cannot run a live web request on paper. The key ideas: a GET always returns a response object, printing that object shows its status code, and reading .text gives the raw page.


      

        
MC 1

        
Which statement about APIs is FALSE?

        
    
          
  • AAPIs let different applications communicate with each other
    • 
          
  • BAPIs define rules and protocols for interacting with software
    • 
          
  • CAPIs are mainly for reading and writing data to local files
    • 
          
  • DAPIs can retrieve data from remote servers and web services
    • 
        

        
        
C is false. APIs are about programs talking to each other over the network, not about local file reading and writing. The other three are all true of APIs.

      

      

        
MC 2

        
Assuming the request succeeds, what does print(requests.get(url)) print?

        
    
          
  • A<Response [404]>
    • 
          
  • B<Response [200]>
    • 
          
  • CA JSON dictionary of the data
    • 
          
  • DThe HTML of the page
    • 
        

        
        
B, <Response [200]>. Printing the response object itself shows its status code, and 200 means success. You would need .text or .json() to see the actual content.

      

      

        
MC 3

        
For a URL that returns a normal HTML web page, what does print(requests.get(url).text[0]) print?

        
    
          
  • AA curly brace, {
    • 
          
  • BThe letter i
    • 
          
  • CA square bracket, [
    • 
          
  • DAn angle bracket, <
    • 
        

        
        
D, an angle bracket. .text is the raw HTML as a string, and an HTML page starts with a tag like <!DOCTYPE> or <html>, so the very first character is <.

      

      

        
MC 4

        
What does r = requests.get(url) do?

        
    
          
  • AReturns a response only if the page holds valid Python string data
    • 
          
  • BReturns a response only if the page holds valid JSON
    • 
          
  • CReturns a response only if the page is a JSON list or dictionary
    • 
          
  • DSends the request and always gets a response object back
    • 
        

        
        
D. A GET always returns a response object, no matter what the page contains. The status code inside it tells you whether the request actually succeeded.
