Start from zero

What a regular expression even is

A regular expression, usually shortened to regex, is a way of describing a shape of text so that Python can go find it for you. Think of it as a search that understands rules, instead of only exact words.

A normal search looks for an exact string. You ask for "cat" and it finds the letters c, a, t. That is useful, but limited. What if you want to find every phone number in a document? Or every word that starts with a capital letter? Or every year from the 1990s? You cannot type out every possibility, there are far too many. A regex lets you describe the rule once ("three digits, then a dash, then four digits") and Python finds every piece of text that fits the rule.

That is the whole idea. You write a pattern (the rule), you hand it to Python along with some text, and Python tells you what matched.

The tool

The re module

All of this lives in a module called re, which is short for "regular expressions." At the top of your program you write import re once, and after that you can use its functions.

Every regex task has the same two ingredients:

  • A pattern, which is the rule you are looking for, written as a string.
  • The text you want to search through, also a string.

You give both to an re function, and it reports back what it found.

The two kinds of characters

Most characters are literal

Here is the gentlest possible starting point. In a pattern, most characters just mean themselves. The pattern cat matches the three letters c, a, t sitting next to each other, nothing more. We call those literal characters.

A small number of characters are special: they stand for an idea instead of themselves. For example, later you will meet \d, which stands for "any digit." We will introduce every special character one at a time, with a plain explanation, before we ever use it. So nothing on this page will appear out of nowhere.

About that little r

You will see patterns written with an r in front, like r"cat" or r"\d". That r makes it a "raw string," which keeps backslashes behaving normally inside the pattern. Exactly why that matters is the first topic of Lecture 8. For now, the rule is simple: always put the r there.

Asking questions about text

The four ways to search

We will use the simplest possible pattern for this whole tab: a plain literal word, with no special symbols at all. That way you can focus on the one new idea here, which is that there are four different functions for searching, and they answer four different questions.

All four take the same two arguments, the pattern and the text. The difference is where they look and what kind of answer they give you back.

FunctionWhere it looksWhat it gives back
re.match(p, s)only at the very START of sone Match object, or None
re.search(p, s)anywhere, stops at the FIRST hitone Match object, or None
re.findall(p, s)everywherea list of the matching strings
re.finditer(p, s)everywherea Match object for each hit

Read that table slowly, because it is the heart of this tab. Two of the functions, match and search, return a single result (or None if there was no match). The other two, findall and finditer, return every result. And within each pair there is one more distinction, explained next.

match versus search

Both return a single match, but they look in different places. match only checks whether the pattern is right at the start of the string. If the string does not begin with the pattern, match gives you None, even if the pattern appears later. search is more forgiving: it scans the whole string and returns the first place the pattern shows up, wherever that is.

findall versus finditer

Both find every match in the whole string. findall hands you a plain list of the matched text, like ['cat', 'cat'], which is what you usually want. finditer hands you the match objects instead, which carry extra information such as the position of each match. You will learn what a match object is in the next tab.

The result you get back

      
What a Match object is

      
When match or search finds something, it does not hand you a plain string. It hands you a small package called a Match object. That object knows several useful facts about the match, and you pull them out by calling methods on it.

      

        
You callYou get

        
m.group()the actual text that matched, as a string

        
m.start()the index in the string where the match begins

        
m.end()the index just AFTER the match ends

        
m.span()a tuple holding both, (start, end)

      

      
The word "index" here means a position in the string, counting from 0. The first character is at index 0, the second at index 1, and so on.

This catches people every time, so lock it in: .span() returns a single tuple (start, end) for the one match you have. It is not a list, and it has nothing to do with "all the matches." The thing that gives you all the matches is findall.

The first special symbols

      
Matching a type of character

      
So far our patterns have been literal words. Now we meet the first real power of regex: instead of one exact character, you can say "any character of a certain type." This is called a character class.

      
Everything in this tab matches exactly one character. Matching more than one is the next tab. Keep that in mind, it is the key to understanding the symbol you asked about earlier.

    


    

      
Your own set

      
Square brackets

      
Put characters inside square brackets and the pattern matches any single one of them. So [aeiou] means "match one character, as long as it is one of these five vowels." You can also use a range with a dash:

      

        
PatternMatches one character that is...

        
[aeiou]one of those vowels (a set you wrote yourself)

        
[a-z]any lowercase letter, a through z

        
[A-Z]any uppercase letter, A through Z

        
[a-zA-Z]any letter, upper or lower

        
[0-9]any digit, 0 through 9

        
[^A-Z]any character that is NOT an uppercase letter

      

      
The ^ has two jobs
Inside square brackets, right at the front, ^ means "not." So [^0-9] means "any character that is not a digit." Outside of brackets, the same ^ means "the start of the string," which you will see in the next tab. Same symbol, two completely different jobs, and the only way to tell them apart is where it sits.

    


    

      
Ready-made sets

      
The shorthand classes

      
Some character classes are so common that regex gives them a one-letter shortcut. Each lowercase shortcut has an uppercase partner that means the exact opposite.

      

        
ShortcutMatches one character that is...The same as

        
\dany digit[0-9]

        
\Dany non-digit[^0-9]

        
\wa "word" character: a letter, digit, or underscore[a-zA-Z0-9_]

        
\Wany non-word character[^a-zA-Z0-9_]

        
\sa whitespace character: space, tab, or newline 

        
\Sany non-whitespace character 

      

      
This is where \d comes from, the symbol from that confusing section earlier. \d on its own matches one digit. So in the string "33", the pattern \d would match the first "3", and separately the second "3", as two single matches. To grab "33" as one whole number, you need to say "one or more digits," which is the next tab.

      
\b is a little different
You will also see \b, the word boundary. Unlike the others, it does not match a character at all, it matches a position, the invisible edge between a word character and a non-word character (or the start or end of the string). You wrap a pattern in \b...\b to insist it be a whole word rather than part of a bigger one. There is an example of this in the next tab.

How many times

      
Repetition

      
Up to now, every pattern matched a fixed number of characters. \d matches exactly one digit. But you usually want a run of characters, like a whole number or a whole word. Repetition symbols handle that. A repetition symbol attaches to the thing right before it and says how many times that thing may repeat.

      

        
SymbolMeans "the thing before me, repeated..."

        
*zero or more times

        
+one or more times

        
?zero or one time (so, optional)

        
{m}exactly m times

        
{m,n}between m and n times

        
{m,}m or more times

      

    


    

      
The symbol you asked about

      
Putting \d and + together

      
Here is the one that confused you, now that both halves make sense:

      
Reading \d+

        
\d means "one digit."

        
+ means "one or more of the thing right before me."

        
So \d+ means "one or more digits in a row," which is exactly how you match a whole number like 12, or 237, or 2025.

      

      
Watch the difference between \d alone and \d+ on the same string. With \d you get single digits. Adding the + tells regex to keep taking digits as long as it can, so each whole number comes out in one piece.

      

        
d_plus.py

        
        

      

      
Regex is greedy
By default, a repetition grabs as much as it possibly can while still letting the rest of the pattern work. That behavior is called greedy. It is why \d+ swallowed the entire "33" instead of stopping after one digit. There is a way to make repetition grab as little as possible instead, and that is the very first topic of Lecture 8.

    


    

      
Position and literal dots

      
Anchors, the dot, and escaping

      
A few more pieces complete the picture. Anchors do not match characters, they pin the pattern to a position in the string.

      

        
SymbolMeans

        
^the start of the string (when it is outside square brackets)

        
$the end of the string

        
.any single character except a newline

        
\.a real, literal period

      

      
The plain dot . is a wildcard, it matches almost any character. So when you actually want to match a period, like the dot in a file name, you put a backslash in front to strip away its special meaning. That is called escaping, and it works for the other special symbols too: \* is a literal asterisk, \? is a literal question mark, and so on.

    


    

      
See them work

      
Reading real patterns out loud

      
The trick to regex is to read a pattern left to right in plain words. Each example below has its reading right above the code.


      
[A-Z][a-z]*
"one uppercase letter, then zero or more lowercase letters," which is the shape of a capitalized word.

      
[A-Za-z]{4,}
"four or more letters in a row," which finds words of length four and up.

      
\b\d{3}\b
"a word boundary, exactly three digits, then a word boundary," which finds three-digit numbers that stand alone, not three digits buried inside a longer number.

      
^[A-Za-z]+  and  [A-Za-z]+$
"letters anchored to the start" gives you the first word, and "letters anchored to the end" gives you the last word.


      

        
repetition.py

        
        

      

      
In the three-digit line, only 487 and 101 come back. 1236 and 1229 are four digits long, so a clean boundary does not fall around exactly three of their digits, and 99 is only two digits, too short.
