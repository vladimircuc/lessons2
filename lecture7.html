Lecture 7: Regular Expressions

A small language for describing patterns in text. By the end you will read these patterns like sentences instead of staring at a wall of symbols.

>>> import re

The idea

What a regular expression is

A regular expression, or regex, is a pattern that describes a shape of text. Instead of asking "is this exact word here," you describe a rule like "three digits in a row" or "a word that starts with a capital letter," and Python finds every piece of text that fits.

You already have string methods like .find(), .split(), and .replace(). Those are great until the rule gets fuzzy. Try finding "every four letter word" or "every year in the 1960s" with plain string methods and you end up with a tangle of loops and conditions. Regex says it in one short pattern.

The tool

The re module

Everything lives in the re module. The rhythm is always the same: you write a pattern, then you ask a question about some text using that pattern. Does it match? Where? How many times?

About that r

You will see patterns written as r"\d+" with an r in front. That is a raw string, and it keeps backslashes literal so \d means what you think it means. We dig into exactly why in the next lecture. For now, just always put the r there.

Four ways to ask

match, search, findall, finditer

These four functions all take a pattern and a string. The difference is what they look for and what they hand back.

FunctionLooksReturns
re.match(p, s)only at the START of sa Match object, or None
re.search(p, s)anywhere, the FIRST hita Match object, or None
re.findall(p, s)everywherea list of all matching strings
re.finditer(p, s)everywhereMatch objects for every hit

The key split

match and search return one Match object (or None). The difference is that match only checks the very start, while search scans the whole string. findall returns a plain list of strings. finditer is like findall but gives you Match objects, so you can get positions too.

The result object

      
What a Match object can tell you

      
When match or search finds something, it hands back a Match object. That object knows four useful things:

      

        
MethodGives you

        
.group()the actual matched text, a string

        
.start()the index where the match begins

        
.end()the index just after the match ends

        
.span()a tuple of (start, end)

The match covers indices 0 through 6, so .start() is 0, .group() is "Georgia", and .end() is 7 (the index just past the last matched character). That makes .span() the tuple (0, 7). The end is always one more than the last index, the same way slicing works.

    


    

      
A common exam trap

      
.span() returns a tuple

      

        
Which statement about Match object methods is INCORRECT?

        
    
          
  • A.group() returns the matched string.
    • 
          
  • B.span() returns a list of every match.
    • 
          
  • C.start() returns the start index of the match.
    • 
          
  • D.end() returns the index after the match.
    • 
        

        
        
B is the incorrect one. .span() returns a single tuple, (start, end), for the one match the object represents. It does not return a list, and it is not about "every match." The list of all matches is what findall gives you.

Sets of characters

      
Character classes

      
Square brackets define a set of characters, and the pattern matches any single character from that set.

      

        
PatternMatches one character that is...

        
[a-z]a lowercase letter

        
[A-Z]an uppercase letter

        
[a-zA-Z]any letter

        
[aeiou]a vowel (your own custom set)

        
[^A-Z]NOT an uppercase letter (the ^ inside means not)

      

      
Two jobs for ^
Inside square brackets at the front, ^ means "not." Outside brackets, ^ means "start of the string." Same symbol, two different jobs depending on where it sits.

    


    

      
Shorthands

      
Predefined classes

      
These shortcuts come up everywhere. The capital-letter versions are the opposite of the lowercase ones.

      

        
ShortcutMatchesSame as

        
\dany digit[0-9]

        
\Dany non-digit[^0-9]

        
\sany whitespacespace, tab, newline

        
\Sany non-whitespacethe opposite of \s

        
\wany word character[a-zA-Z0-9_]

        
\Wany non-word character[^a-zA-Z0-9_]

        
\ba word boundarythe edge of a word (matches a position, not a character)

      

      
\b is special
\b does not match a character, it matches a position, the edge between a word character and a non-word character. You wrap a pattern in \b...\b to say "this has to be a whole word," not part of a bigger one.

How many

      
Repetition

      
A repetition symbol applies to the thing right before it, and says how many times that thing may repeat.

      

        
SymbolMeans

        
*0 or more

        
+1 or more

        
?0 or 1 (optional)

        
{m}exactly m

        
{m,n}between m and n

        
{m,}m or more

      

      
Regex is greedy
By default a repetition grabs as much as it possibly can while still letting the rest of the pattern match. That is called greedy matching. There is a way to make it grab as little as possible instead, and that is the first thing we cover next lecture.

    


    

      
Position

      
Anchors, the dot, and escaping

      

        
SymbolMeans

        
^start of the string (outside brackets)

        
$end of the string

        
.any single character except a newline

        
\.a literal dot (the backslash escapes it)

      

      
Escaping
Symbols like ., *, and ? are special. When you want the literal character instead of its special meaning, put a backslash in front. So \. matches an actual period.

Given On the exam reference sheet

        
You will be handed these two tables. Know how to read them fast, but do not burn crib space on them.

        
PREDEFINED CHARACTER CLASSES
  \d   any digit            [0-9]
  \D   any non-digit        [^0-9]
  \s   any whitespace       space, tab, newline ...
  \S   any non-whitespace
  \w   any word char        [a-zA-Z0-9_]
  \W   any non-word char    [^a-zA-Z0-9_]

META CHARACTERS
  .    any character except newline
  \    escape a special character
  |    or  (rarely used)
  ^    start of string  (or "not" inside [ ])
  $    end of string
  *    0 or more of the previous
  +    1 or more of the previous
  ?    0 or 1 of the previous
  { }  bounded repetition,  {m}  {m,n}  {m,}
  [ ]  a character class
  ( )  a capture group

      


      

        
Memorize Put this on your crib sheet

        
What each function returns (this is NOT on the sheet):

        
re.match(p, s)     start only   -> Match object or None
re.search(p, s)    first hit    -> Match object or None
re.findall(p, s)   all hits     -> list of strings
re.finditer(p, s)  all hits     -> Match objects (positions)

        
Match object methods:

        
m.group()  -> the matched text (a string)
m.start()  -> start index
m.end()    -> index just AFTER the match
m.span()   -> a TUPLE (start, end)   not a list

        
\b is NOT on the given sheet. It matches a word boundary (a position at the edge of a word). Wrap a pattern in \b...\b to force a whole word.

        
Repetition is greedy by default (grabs as much as it can). Always write patterns as raw strings, r"...".

        
Ready-made recipes:

        
\d{3}              exactly three digits
\d+                a whole number
[A-Za-z]+   or  \w+   a word
[A-Z][a-z]*        a capitalized word
^[A-Za-z]+         the first word
[A-Za-z]+$         the last word
[A-Za-z]{4,}       words of 4+ letters
\.                 a literal period
