How to use it

For each question, answer in your own words first, no peeking. Then we open the model answer to compare, and mark how it went: red (blank / wrong), amber (partial), green (solid). Recall is harder than recognition on purpose, so amber is normal and fine. The colors build a map of which objectives to drill next.

Domain 1 · Networking Concepts — 23% of the exam. The vocabulary and logic of how networks work: models, devices, cloud, ports, media, topologies, and IP addressing. This is the base everything else stands on, and it is where subnetting lives.

12 questions · objectives 1.1–1.8

Domain 2 · Network Implementation — 20% of the exam. Actually building the network: routing, switching, wireless, and the physical install. More configuration-flavored, so expect scenario questions.

10 questions · objectives 2.1–2.4

Domain 3 · Network Operations — 19% of the exam. Keeping a running network healthy: documentation, monitoring, disaster recovery, IP services, and how you remotely access and manage gear.

10 questions · objectives 3.1–3.5

Domain 4 · Network Security — 14% of the exam. The smallest domain, but still points: core security principles, the common attack types, and the defenses you put in place to stop them.

6 questions · objectives 4.1–4.3

Domain 5 · Network Troubleshooting — 24% of the exam, the largest. The payoff domain: a repeatable methodology, plus diagnosing cabling, services, and performance, and knowing which tool to reach for. Heavily scenario-based.

12 questions · objectives 5.1–5.5
