Python & Data Diagnostic

A warm-up check across plain Python, NumPy, and pandas. Work through one task at a time. The goal is not to finish everything, it is to see how far you can comfortably get. Skip a task if you are stuck and move on.

Easy Medium Harder
How to use this
  1. For each problem, copy the Data setup block into a fresh cell in your Jupyter notebook and run it.
  2. Before doing any task, look at the data: print it, check its shape, see what columns or keys it has.
  3. Do the tasks one at a time in their own cells. Try it yourself first, the solutions are collapsed at the bottom of each problem.

1 Student Records plain Python

A list of dictionaries, no libraries needed. This is pure Python: loops, conditionals, and a small function.

Data setup, copy this into a cell
students = [
    {"name": "Ana",   "grade": 88, "age": 16},
    {"name": "Ben",   "grade": 72, "age": 17},
    {"name": "Cara",  "grade": 95, "age": 16},
    {"name": "Dan",   "grade": 61, "age": 18},
    {"name": "Eli",   "grade": 78, "age": 17},
    {"name": "Faye",  "grade": 84, "age": 16},
    {"name": "Gus",   "grade": 90, "age": 17},
    {"name": "Hana",  "grade": 55, "age": 18},
]
Tasks
  1. Easy Print how many students are in the list.
  2. Easy Print just the name of every student, one per line.
  3. Easy Compute and print the average grade of all students.
  4. Medium Print the names of all students who scored 80 or higher.
  5. Medium Find and print the student with the highest grade (name and grade).
  6. Medium Count how many students are in each age, e.g. {16: 3, 17: 3, 18: 2}.
  7. Harder Write a function letter(grade) that returns "A" for 90+, "B" for 80 to 89, "C" for 70 to 79, else "F". Use it to print each name with its letter.
Solutions 
# Task 1 - basic len(), do they know lists have a length
print(len(students))

# Task 2 - simple loop + dict access by key
for s in students:
    print(s["name"])

# Task 3 - accumulate a total then divide; tests loop + arithmetic
total = 0
for s in students:
    total += s["grade"]
print(total / len(students))
# (a stronger student may use: sum(s["grade"] for s in students) / len(students))

# Task 4 - filtering with a condition inside a loop
for s in students:
    if s["grade"] >= 80:
        print(s["name"])

# Task 5 - tracking a "best so far"; tests comparison logic
best = students[0]
for s in students:
    if s["grade"] > best["grade"]:
        best = s
print(best["name"], best["grade"])
# (stronger: max(students, key=lambda s: s["grade"]))

# Task 6 - building a dict by counting; classic beginner stumbling point
counts = {}
for s in students:
    age = s["age"]
    counts[age] = counts.get(age, 0) + 1
print(counts)

# Task 7 - defining and calling a function with branching logic
def letter(grade):
    if grade >= 90:
        return "A"
    elif grade >= 80:
        return "B"
    elif grade >= 70:
        return "C"
    else:
        return "F"

for s in students:
    print(s["name"], letter(s["grade"]))

What this reveals: are the Python fundamentals actually rusty or solid? Watch tasks 5 to 7. Tracking "best so far", counting into a dict, and writing a branching function are the three places beginners stall. If he reaches for max(key=...) or a comprehension on his own, he is past beginner.

2 Order Log plain Python

A second pure-Python problem, no libraries, but testing different muscles: strings, nested structures, and set logic. Useful if NumPy and pandas turn out to be new to him.

Data setup, copy this into a cell
# Each order: customer, and the items they bought
orders = [
    ("ana",  ["apple", "bread", "milk"]),
    ("ben",  ["milk", "eggs"]),
    ("ana",  ["bread", "eggs", "cheese"]),
    ("cara", ["apple", "milk"]),
    ("ben",  ["bread", "milk", "apple"]),
    ("ana",  ["milk"]),
]
Tasks
  1. Easy Print how many orders there are, and how many items are in the very first order.
  2. Easy Print each customer's name in uppercase, one per line.
  3. Medium Print the total number of items sold across all orders (count every item, including repeats).
  4. Medium Print the set of unique items that appear anywhere (no duplicates).
  5. Medium Build a dict mapping each customer to how many orders they placed, e.g. {"ana": 3, "ben": 2, "cara": 1}.
  6. Harder How many times was "milk" ordered in total across all orders?
  7. Harder Build a dict mapping each customer to the set of all distinct items they have ever bought.
Solutions 
# Task 1 - len() on the list, and len() on an item inside a tuple
print(len(orders))
print(len(orders[0][1]))   # orders[0] is a tuple; [1] is its item list

# Task 2 - string method .upper(); unpacking the tuple in the loop
for name, items in orders:
    print(name.upper())

# Task 3 - a running total across nested lists (loop inside a loop, or sum of lens)
total = 0
for name, items in orders:
    total += len(items)
print(total)
# (stronger: sum(len(items) for name, items in orders))

# Task 4 - sets remove duplicates; building one up
unique = set()
for name, items in orders:
    for item in items:
        unique.add(item)
print(unique)

# Task 5 - counting into a dict, but keyed on the customer this time
order_counts = {}
for name, items in orders:
    order_counts[name] = order_counts.get(name, 0) + 1
print(order_counts)

# Task 6 - counting a specific value across nested data
milk = 0
for name, items in orders:
    milk += items.count("milk")   # .count on a list
print(milk)

# Task 7 - dict of sets; the hardest, combines dict-building + set logic
bought = {}
for name, items in orders:
    if name not in bought:
        bought[name] = set()
    bought[name].update(items)   # .update adds all items to the set
print(bought)
# (stronger: bought.setdefault(name, set()).update(items))

What this reveals: different skills than Problem 1 on purpose. Task 1 tests whether tuple/list nesting confuses him. Task 4 checks if he knows sets exist for deduping (beginners often write clumsy "if not in list" loops instead). Tasks 5 and 7 are the ceiling here, dict-building and especially a dict-of-sets. If both Problem 1 and this feel easy to him, his core Python is solid and you can move quickly into NumPy/pandas.

3 Sensor Readings NumPy

Daily temperature readings as a NumPy array. Tests indexing, slicing, boolean masks, and aggregations.

Data setup, copy this into a cell
import numpy as np

# 14 days of temperature readings (Celsius)
temps = np.array([20.1, 21.5, 19.8, 22.0, 23.4, 18.9, 25.1,
                  24.6, 19.2, 20.8, 26.3, 22.7, 17.5, 21.9])
Tasks
  1. Easy Print the shape of temps and how many readings there are.
  2. Easy Print the mean, minimum, and maximum temperature.
  3. Easy Print the first 7 readings (the first week) using slicing.
  4. Medium Print all readings that are above 22 degrees, using a boolean mask.
  5. Medium Count how many days were above 22 degrees.
  6. Medium Convert the whole array to Fahrenheit (C * 9/5 + 32) in one line, no loop.
  7. Harder Find the index of the hottest day, then print which week it fell in (week 1 = first 7 days, week 2 = last 7).
Solutions 
# Task 1 - .shape vs len(); do they know arrays have attributes
print(temps.shape)
print(len(temps))

# Task 2 - built-in aggregation methods
print(temps.mean(), temps.min(), temps.max())

# Task 3 - slicing, same syntax as lists
print(temps[:7])

# Task 4 - boolean masking, THE core numpy idea to check for
print(temps[temps > 22])

# Task 5 - a mask is booleans; summing booleans counts True values
print((temps > 22).sum())
# (also valid: np.sum(temps > 22) )

# Task 6 - vectorized arithmetic, no loop needed
print(temps * 9/5 + 32)

# Task 7 - argmax + a little logic
i = temps.argmax()
print("hottest index:", i)
print("week 1" if i < 7 else "week 2")

What this reveals: the whole point of NumPy is vectorization, doing things to the whole array at once without a loop. Task 4 (boolean masks), task 5 (summing a mask), and task 6 (vectorized math) are the tell. If he writes a for loop for task 6, he knows Python but not the NumPy mindset yet, which is fine and tells you exactly where to teach.

4 Sales Table pandas

A small sales dataset loaded into a DataFrame. Tests selecting, filtering, grouping, and a missing value.

Data setup, copy this into a cell
import pandas as pd
import io

data = """name,region,sales,units
Ana,East,250,12
Ben,West,180,9
Cara,East,320,15
Dan,West,,7
Eli,East,210,11
Faye,West,275,14
Gus,East,190,8
Hana,West,300,16"""

df = pd.read_csv(io.StringIO(data))
Tasks
  1. Easy Show the first few rows, the column names, and the shape of df.
  2. Easy Print just the sales column.
  3. Easy Print the average and total of the sales column.
  4. Medium One row has a missing sales value. Find it, then fill the missing value with the average sales.
  5. Medium Print only the rows where region is "East".
  6. Medium Use groupby to print total sales per region.
  7. Harder Add a new column per_unit = sales divided by units, then print the name with the highest per-unit value.
Solutions 
# Task 1 - the "look at your data first" habit
print(df.head())
print(df.columns)
print(df.shape)

# Task 2 - selecting a single column
print(df["sales"])

# Task 3 - column aggregation methods; note NaN is skipped automatically
print(df["sales"].mean(), df["sales"].sum())

# Task 4 - spotting and handling missing data
print(df[df["sales"].isna()])          # find the missing row
avg = df["sales"].mean()
df["sales"] = df["sales"].fillna(avg)  # fill it
print(df)

# Task 5 - boolean filtering of rows, the pandas version of a mask
print(df[df["region"] == "East"])

# Task 6 - groupby, the single most important pandas skill for data work
print(df.groupby("region")["sales"].sum())

# Task 7 - creating a derived column, then finding a max
df["per_unit"] = df["sales"] / df["units"]
top = df.loc[df["per_unit"].idxmax()]
print(top["name"], round(top["per_unit"], 2))

What this reveals: task 1 shows whether he inspects data before touching it (a real habit, not a beginner reflex). Task 4 is missing-data awareness. Task 6 (groupby) is the big one, it is the heart of almost all real data analysis. If he gets through groupby and the derived column in task 7, he is more than ready for the project work.

5 Putting It Together pandas + NumPy

Load a small dataset, clean it, and pull out a simple insight. This is the flavor of what early AI/ML prep work feels like, no modeling yet.

Data setup, copy this into a cell
import pandas as pd
import numpy as np
import io

data = """hours_studied,attendance,score
2,60,55
5,80,70
1,50,48
8,95,88
4,75,66
7,90,82
3,65,60
6,85,79
9,98,91
2,55,52"""

study = pd.read_csv(io.StringIO(data))
Tasks
  1. Easy Inspect the data: shape, column names, and the summary stats from describe().
  2. Easy Print the average score across all rows.
  3. Medium Add a column passed that is True when score is 65 or higher, else False.
  4. Medium How many students passed? What fraction of the class is that?
  5. Medium Print the average hours studied for students who passed vs students who did not.
  6. Harder Use .corr() to see how strongly hours_studied relates to score. Print just that one correlation number.
  7. Harder In words (a printed string is fine), what does that correlation tell you? Would hours_studied be a useful feature for predicting score?
Solutions 
# Task 1 - describe() is the fast way to size up a dataset
print(study.shape)
print(study.columns)
print(study.describe())

# Task 2 - simple column mean
print(study["score"].mean())

# Task 3 - a boolean column from a condition (vectorized, no loop)
study["passed"] = study["score"] >= 65
print(study)

# Task 4 - summing a boolean column counts the Trues
n = study["passed"].sum()
print(n, "passed")
print(n / len(study))          # the fraction

# Task 5 - filter then aggregate, combining two earlier skills
print(study[study["passed"]]["hours_studied"].mean())
print(study[~study["passed"]]["hours_studied"].mean())  # ~ means "not"

# Task 6 - correlation; the .loc just pulls out the single number
print(study["hours_studied"].corr(study["score"]))

# Task 7 - interpretation, no single right code answer
print("Correlation is very high (close to 1), so more hours studied"
      " strongly tracks with higher scores. It would be a useful feature"
      " for predicting score.")

What this reveals: this is the ceiling probe. Task 5 chains filtering + aggregation, task 6 introduces correlation (his first taste of "which inputs predict the output", the seed of feature thinking in ML). Task 7 is not about code at all, it is whether he can reason about data. A student who can explain task 7 in his own words is ready to start the real project. If he stalls around task 3 or 4, you start the curriculum earlier and build up.

Diagnostic warm-up. Difficulty is intentionally capped at beginner-to-early-intermediate. The aim is to find the ceiling, not to trip anyone up.
